# 兼容性验证矩阵

每个场景都先在旧工程取得基线，再在新工程重复；记录可观察结果、协议帧和必要的
寄存器/波形证据。

| 场景 | 输入 | 必查输出 |
| --- | --- | --- |
| 冷启动 | 完全断电后上电 | Boot 判定、GPIO 初始电平、App 进入、首个任务/上报 |
| 正常开机 | 电源键手势 | 通道顺序、PC 供电、LED/蜂鸣器时序、模式变化 |
| 急停 | 按下与释放 | 状态帧、其他板语音/联动、屏蔽与解除时机 |
| 充电 | 接入、拔出、温度边界 | 电压判定、充电电流、加热、SN/状态上报 |
| UART/CAN 请求 | 每个支持的远程帧 | 地址、RTR、DLC、回应帧、超时、周期上报 |
| IAP | 进入、擦除、写入、执行、确认 | 标志、帧顺序、跳转、失败回退 |
| 看门狗 | 等待或受控触发 | Reset cause、一次性事件、是否错误循环重启 |
| 关机/低功耗 | 命令、按键、低 SOC | 上报顺序、BMS 输出、ShipMode、电流状态 |

## 时序记录

对肉眼或上位机可见的行为，记录开始条件、首个边沿、周期、结束条件和容差。不要只
比较“看起来差不多”；LED、蜂鸣器、协议重试与看门狗均需要明确单位。

## 判定

- **通过**：行为和协议与旧工程一致，或存在用户批准且记录在案的差异。
- **待确认**：只有源码推断，没有硬件/抓帧证据。
- **失败**：状态、电平、时序、协议字段或恢复路径不一致。
